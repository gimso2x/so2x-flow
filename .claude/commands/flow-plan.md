# /flow-plan

Use this command to create a plan or task document without implementation.

## Required flow
1. Read relevant docs.
2. Create a task or plan document.
3. Produce planning output only.
4. Do not invoke implementation behavior.

## Difference from `/flow-init`
- `/flow-init` bootstraps the whole workspace.
- `/flow-plan` creates planning artifacts for a specific request.

## Rules
- No code changes.
- No implementation role.
- Keep decomposition explicit and reviewable.
